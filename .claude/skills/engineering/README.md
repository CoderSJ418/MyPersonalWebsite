# 工程工具

我日常用于代码工作的 skills。

- **[diagnose](./diagnose/SKILL.md)** — 硬 bug 和性能回归的有纪律诊断循环：复现 → 最小化 → 假设 → 检测 → 修复 → 回归测试。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 审视会议，针对现有领域模型挑战你的计划，锐化术语，并实时更新 `CONTEXT.md` 和 ADR。
- **[triage](./triage/SKILL.md)** — 通过分类角色的状态机对 issue 进行分类。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 在 `CONTEXT.md` 中的领域语言和 `docs/adr/` 中的决策指导下找到代码库的深化机会。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 搭建每个仓库的配置（问题跟踪器、分类标签词汇、领域文档布局），供其他工程 skills 使用。
- **[tdd](./tdd/SKILL.md)** — 使用红绿重构循环的测试驱动开发。一次垂直切片地构建功能或修复 bug。
- **[to-issues](./to-issues/SKILL.md)** — 使用垂直切片将任何计划、规格或 PRD 分解为可独立处理的 GitHub issue。
- **[to-prd](./to-prd/SKILL.md)** — 将当前对话上下文转换为 PRD 并作为 GitHub issue 提交。
- **[zoom-out](./zoom-out/SKILL.md)** — 告诉 agent 缩小视图并对不熟悉的代码部分提供更广阔的上下文或更高层次的视角。
- **[prototype](./prototype/SKILL.md)** — 构建一次性原型来充实设计 — 针对状态/业务逻辑问题的可运行终端应用，或多个从一条路由切换的完全不同的 UI 变体。
