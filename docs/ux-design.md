**项目名称**：个人网站（前端技能展示平台）
**UX设计师**：Sally（BMad UX Expert）
**设计日期**：2026年1月19日
**项目版本**：v1.0
**状态**：待评审

---

## 📋 执行摘要

本用户体验设计基于需求分析和技术架构，为个人网站项目提供完整的UX设计方案。设计遵循用户中心设计原则，强调直观性、流畅性和可访问性，确保访问者能够快速了解用户的技术实力和项目经验。

**核心设计原则**：
- **用户中心**：以用户需求为出发点
- **简洁直观**：清晰的导航和信息架构
- **流畅交互**：流畅的过渡和微交互
- **可访问性**：支持键盘导航和屏幕阅读器
- **响应式设计**：适配所有设备

**关键设计决策**：
- 移动端优先的响应式设计
- 清晰的信息架构和导航
- 流畅的动画和微交互
- 暗黑模式支持
- 优秀的SEO和可访问性

## 1. 用户画像与场景

### 1.1 用户画像

#### 画像1：HR（人力资源）
- **角色**：HR招聘专员
- **目标**：快速了解候选人的基本信息和技能匹配度
- **痛点**：时间有限，需要快速筛选候选人
- **需求**：
  - 清晰的个人介绍
  - 技能标签化展示
  - 项目经验时间线
  - 便捷的联系方式

#### 画像2：技术面试官
- **角色**：前端技术负责人
- **目标**：深入了解候选人的技术能力和项目经验
- **痛点**：需要评估技术深度和工程化能力
- **需求**：
  - 技术栈分类展示
  - 技能熟练度可视化
  - 项目技术难点描述
  - 代码质量展示

#### 画像3：技术社区同行
- **角色**：前端开发者
- **目标**：学习技术经验，建立联系
- **痛点**：寻找高质量的技术内容
- **需求**：
  - 技术博客文章
  - 开源项目链接
  - 技术栈选择理由
  - 社交媒体链接

#### 画像4：潜在雇主
- **角色**：公司CTO/技术总监
- **目标**：评估候选人是否适合团队
- **痛点**：需要全面了解候选人的能力和潜力
- **需求**：
  - 技术能力匹配
  - 项目经验相关性
  - 团队协作能力
  - 快速上手能力

### 1.2 用户场景

#### 场景1：HR快速浏览
**用户故事**：作为HR，我需要在30秒内了解这个候选人的基本信息和技能匹配度，以便决定是否继续深入了解。

**用户流程**：
1. 打开网站 → 看到Hero区域（职业定位+核心能力）
2. 滚动查看技术栈 → 看到技能标签
3. 查看精选项目 → 了解项目经验
4. 点击联系方式 → 发送邮件或下载简历

**设计要点**：
- Hero区域突出职业定位
- 技术栈使用图标快速识别
- 项目卡片简洁明了
- 联系方式显眼易用

#### 场景2：技术面试官深度了解
**用户故事**：作为技术面试官，我需要深入了解候选人的技术栈、项目经验和技术深度，以便评估技术能力。

**用户流程**：
1. 打开网站 → 查看技术栈分类
2. 点击技能展示 → 查看技能熟练度和深度
3. 浏览项目详情 → 了解技术难点和解决方案
4. 阅读技术博客 → 了解技术思考和学习能力

**设计要点**：
- 技能分类清晰，易于查找
- 技能详情展示深度
- 项目详情包含技术栈和难点
- 博客文章代码高亮

#### 场景3：技术社区同行学习
**用户故事**：作为前端开发者，我需要学习技术经验，查看开源项目，建立联系。

**用户流程**：
1. 打开网站 → 浏览技术博客
2. 点击博客文章 → 阅读技术内容
3. 查看项目 → 了解开源项目
4. 点击社交媒体 → 建立联系

**设计要点**：
- 博客文章易于阅读
- 代码高亮清晰
- 项目链接直达
- 社交媒体链接完整

#### 场景4：移动端浏览
**用户故事**：作为移动端用户，我需要在手机上快速浏览网站，获取关键信息。

**用户流程**：
1. 打开网站 → 看到移动端优化的Hero区域
2. 点击菜单 → 展开导航菜单
3. 滑动浏览 → 流畅的滚动体验
4. 点击联系 → 快速联系

**设计要点**：
- 移动端优先设计
- 导航菜单可折叠
- 触摸交互优化
- 加载速度快

#### 3.1.2 Hero区域设计

**视觉设计**：
- 背景：渐变色或粒子效果
- 布局：居中对齐
- 字体：大标题 + 副标题

**内容元素**：
- 头像/Logo：圆形或方形，带阴影
- 姓名：大标题（H1）
- 职业定位：副标题（H2）
- 核心能力：简短描述（2-3行）
- CTA按钮：主按钮（查看项目）+ 次按钮（联系我）

**交互设计**：
- 打字机效果：职业定位文字逐字显示
- 悬浮效果：头像轻微浮动
- 按钮动画：鼠标悬浮时放大

**响应式设计**：
- 桌面端：左右布局（头像左，文字右）
- 平板端：上下布局（头像上，文字下）
- 移动端：垂直居中布局

#### 3.1.3 技术栈概览设计

**视觉设计**：
- 布局：网格布局（3-4列）
- 图标：圆形或方形图标
- 颜色：品牌色或灰色系

**内容元素**：
- 技术栈图标：Vue、React、TypeScript等
- 技术栈名称：图标下方文字
- 分类标签：前端框架、工具、语言等

**交互设计**：
- 悬浮效果：图标放大，显示详情
- 点击跳转：跳转到技能页面

**响应式设计**：
- 桌面端：4列网格

## 4. 交互设计

### 4.1 导航交互

**主导航**：
- 桌面端：水平导航栏
- 移动端：汉堡菜单，点击展开
- 当前页面高亮：导航项高亮显示

**面包屑导航**：
- 位置：页面顶部
- 点击跳转：点击面包屑跳转

**返回顶部**：
- 位置：页面右下角
- 显示条件：滚动超过500px时显示
- 点击行为：平滑滚动到顶部

### 4.2 按钮交互

**主按钮**：
- 样式：品牌色背景，白色文字
- 悬浮效果：背景色加深，轻微放大
- 点击效果：按下缩小
- 禁用状态：灰色背景，不可点击

**次按钮**：
- 样式：透明背景，品牌色边框
- 悬浮效果：背景色淡入

**链接按钮**：
- 样式：无背景，品牌色文字
- 悬浮效果：下划线动画

### 4.3 卡片交互

**项目卡片**：
- 悬浮效果：卡片上浮，阴影加深
- 点击效果：跳转到详情页
- 图片放大：图片轻微放大

**博客卡片**：
- 悬浮效果：卡片上浮，阴影加深
- 点击效果：跳转到详情页

### 4.4 表单交互

**输入框**：
- 聚焦效果：边框高亮
- 验证错误：红色边框，错误提示
- 验证成功：绿色边框

**提交按钮**：
- 提交中：加载动画，禁用按钮
- 提交成功：成功提示，重置表单
- 提交失败：错误提示，保留表单

### 4.5 动画效果

**页面过渡**：
- 页面切换：淡入淡出
- 元素进入：从下往上淡入

**滚动动画**：
- 元素进入：滚动到可视区域时淡入
- 阅读进度：顶部进度条更新

**悬浮动画**：
- 按钮：轻微放大
- 卡片：上浮，阴影加深
- 图标：旋转或放大

## 5. 响应式设计

### 5.1 断点设计

**断点定义**：
- 移动端：< 640px
- 平板端：640px - 1024px
- 桌面端：> 1024px

### 5.2 移动端设计

**布局调整**：

- 字体：适当减小字体大小

**交互优化**：
- 触摸目标：最小44x44px
- 手势支持：滑动、缩放
- 键盘优化：虚拟键盘适配

### 5.3 平板端设计

**布局调整**：

### 5.4 桌面端设计

**布局调整**：

- 字体：标准字体大小

## 6. 可访问性设计

### 6.1 键盘导航

**Tab键导航**：
- 导航顺序：从上到下，从左到右
- 焦点可见：清晰的焦点样式
- 跳过导航：跳到主内容按钮

**快捷键**：
- Esc：关闭模态框
- Enter：提交表单
- Space：切换开关

### 6.2 屏幕阅读器支持

**语义化HTML**：
- 使用语义化标签：header、nav、main、footer
- ARIA标签：aria-label、aria-describedby
- Alt文本：图片描述

**标题层次**：
- H1：页面标题（每页一个）
- H2：主要章节
- H3-H6：子章节

### 6.3 颜色对比度

**对比度要求**：
- 文本：至少4.5:1
- 大文本：至少3:1
- 图标：至少3:1

**色盲友好**：
- 不仅依赖颜色传达信息
- 使用图标和文字配合

### 6.4 字体大小

**最小字体**：
- 正文：至少16px
- 标题：根据层级递增

**字体缩放**：
- 支持浏览器字体缩放
- 使用相对单位（rem、em）
