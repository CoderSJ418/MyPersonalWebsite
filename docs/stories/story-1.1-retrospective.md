## 🐛 遇到的问题和解决方案

### 问题 1: PowerShell 命令兼容性
**严重程度**: 中  
**影响范围**: 测试验证阶段

**问题描述**:
```powershell
cd E:\work\AI\MyPersonalWebsite && npm run test
# 错误: The token '&&' is not a valid statement separator
```

**根本原因**: PowerShell 5.1 不支持 `&&` 命令分隔符

**解决方案**:
```powershell
# 使用分号替代
cd E:\work\AI\MyPersonalWebsite; npm run test
```

**预防措施**: 记录到 `error_patterns.md`，作为开发规则

---

### 问题 2: Shell 命令超时限制
**严重程度**: 低  
**影响范围**: 测试验证阶段

**问题描述**:
```javascript
run_shell_command({ command: '...', timeout_ms: 72000 })
// 错误: Timeout cannot exceed 3600 seconds
```

**根本原因**: run_shell_command 工具限制超时最大值为 3600 秒

**解决方案**:
```javascript
// 不设置超时参数，使用默认值
run_shell_command({ command: '...' })
```

**预防措施**: 记录到 `error_patterns.md`

---

### 问题 3: 测试污染
**严重程度**: 中  
**影响范围**: 测试验证阶段

**问题描述**: 运行完整测试套件时，其他测试文件的 Vue 警告影响新测试的验证

**根本原因**: themeStore.spec.ts 中存在 Vue 生命周期警告

**解决方案**:
```bash
# 单独运行目标组件测试
npx vitest run tests/unit/components/BlogList.spec.ts
```

**预防措施**: 测试新组件时，单独运行该组件的测试文件

---

### 问题 4: ESLint 误报
**严重程度**: 低  
**影响范围**: 代码质量检查

**问题描述**: `props` 变量被标记为未使用

**根本原因**: props 在 `<script setup>` 中定义，只在模板中使用，ESLint 误报

**解决方案**:
```typescript
// eslint-disable-next-line @typescript-eslint/no-unused-vars
const props = defineProps<Props>()
```

**预防措施**: 记录到 `error_patterns.md`

---

### 问题 5: BlogCard 组件缺少分类显示
**严重程度**: 高  
**影响范围**: 功能完整性

**问题描述**: BlogCard 组件未显示文章分类，导致 SC-2 验收标准未完全通过

**根本原因**: 开发过程中遗漏了分类显示功能

**解决方案**:
```vue
<!-- 添加分类显示 -->
<div class="mb-3 md:mb-4">
  <button
    class="px-2 py-1 bg-theme-primary/20 text-theme-primary rounded text-xs font-medium"
    @click.stop="handleCategoryClick(post.category)"
  >
    {{ post.category }}
  </button>
</div>
```

**预防措施**: 
- 审查阶段仔细检查所有验收标准
- 开发过程中对照验收标准逐项检查

---

### 问题 6: BlogList 组件缺少单元测试
**严重程度**: 中  
**影响范围**: 测试覆盖率

**问题描述**: BlogList 组件未创建单元测试，导致测试覆盖率不足

**根本原因**: 开发过程中遗漏了测试文件

**解决方案**:
- 创建 `tests/unit/components/BlogList.spec.ts`
- 编写 21 个测试用例，覆盖所有功能
- 测试结果：21/21 通过 ✅

**预防措施**:
- 在 Task 列表中明确标记测试任务
- 开发完成后立即检查测试覆盖率

---

### 问题 7: TypeScript 类型检查工具失败
**严重程度**: 低  
**影响范围**: 代码质量检查

**问题描述**: `vue-tsc` 工具报错 "Search string not found"

**根本原因**: vue-tsc 版本兼容性问题

**解决方案**:
```bash
# 使用 --noEmit 参数
vue-tsc --noEmit
```

**预防措施**: 记录到 `error_patterns.md`

## 🎓 经验教训

### 技术层面

1. **Windows PowerShell 兼容性**
   - 必须使用分号 `;` 替代 `&&`
   - 记录到开发规则文档

2. **Vue 3 Composition API 特性**
   - props 变量在 `<script setup>` 中只在模板使用时需要忽略 ESLint
   - 使用 `eslint-disable-next-line` 注释

3. **测试隔离原则**
   - 测试新组件时单独运行该组件的测试文件
   - 避免其他测试文件的干扰

4. **TypeScript 类型检查**
   - 使用 `vue-tsc --noEmit` 参数
   - 或升级到最新版本

### 流程层面

1. **开发阶段对照验收标准**
   - 逐项检查，避免遗漏
   - 提高一次性通过率

2. **测试驱动开发**
   - 先编写测试用例
   - 再实现功能
   - 确保覆盖率达标

3. **代码审查前置**
   - 开发过程中自我审查
   - 减少审查阶段的修复时间

### 团队层面

1. **错误模式总结**
   - 记录到 `error_patterns.md`
   - 供后续参考

2. **最佳实践分享**
   - 记录到 `experience/` 目录
   - 提高团队整体水平
