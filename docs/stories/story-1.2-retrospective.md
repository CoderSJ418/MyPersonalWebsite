## 🐛 遇到的问题和解决方案

### 问题 1: 测试类名不匹配

**现象**: 单元测试失败，因为测试中使用的类名与实际组件的类名不一致

**根本原因**:
- 在重构或修改组件类名时，没有同步更新测试代码
- 测试编写者可能基于假设或旧代码编写测试
- 缺乏自动化检查机制

**具体表现**:
- RelatedPosts: 测试使用 `.related-posts__card`，组件使用 `.related-posts__item`
- PostNavigation: 测试使用 `.post-nav__prev`，组件使用 `.post-navigation__item--prev`

**解决方案**:
- 更新所有测试文件中的类名，使其与实际组件一致
- 添加 `await router.isReady()` 等待路由导航完成
- 删除错误的类名检查（如检查 `transition` 类名）

**经验教训**:
- 建立严格的代码审查流程，确保测试与组件代码同步更新
- 在编写测试前，仔细检查组件的实际实现
- 使用组件开发工具验证实际的 DOM 结构

### 问题 2: 路由导航测试失败

**现象**: 点击导航后 `router.currentRoute.value.name` 是 undefined

**根本原因**:
- 测试编写者对 Vue Router 的异步特性理解不够深入
- 测试代码没有正确处理异步操作
- 缺乏 Vue Router 测试的最佳实践

**解决方案**:
- 使用 `await router.isReady()` 等待路由导航完成
- 等待 Promise 完成而不是立即检查路由状态

**经验教训**:
- 充分理解 Vue Router 的异步特性
- 遵循 Vue Router 测试的最佳实践
- 对异步操作使用适当的等待机制

### 问题 3: 博客数据缺少 category 字段

**现象**: 相关文章推荐功能无法正常工作

**根本原因**:
- `blog-index.json` 中的文章数据缺少 `category` 字段
- 组件依赖 `category` 字段来推荐相关文章

**解决方案**:
- 为所有 15 篇文章添加 `category` 字段
- 根据文章内容分配合适的分类（Vue3、TypeScript、构建工具、性能优化、面试、学习路线、教程、思考等）

**经验教训**:
- 确保数据结构的完整性
- 在开发组件时考虑数据依赖关系
- 定期检查数据与组件的兼容性

### 问题 4: CSS 变量定义问题

**现象**: 亮色模式下文字颜色不正确

**根本原因**:
- `:root` 中的 CSS 变量使用了暗黑模式的值
- 导致亮色模式下文字颜色为白色，在白色背景上看不见

**解决方案**:
- 修正 `main.css` 中的 CSS 变量定义
- `:root` 使用亮色模式值
- `.dark` 使用暗黑模式值

**经验教训**:
- 确保默认主题（亮色模式）的样式正确
- 使用 CSS 变量时考虑主题切换
- 测试不同主题下的显示效果

### 问题 5: Vitest UI 服务器冲突

**现象**: 多次运行 `npx vitest` 命令导致多个 Vitest UI 服务器同时启动，造成端口冲突和 WebSocket 连接失败

**根本原因**:
- 使用 `npx vitest` 命令会启动 UI 服务器
- 多次运行会启动多个服务器，占用相同端口
- 没有使用 `--run` 选项直接运行测试

**解决方案**:
- 使用 `npx vitest run` 命令直接运行测试，不启动 UI
- 如果需要使用 UI，先关闭所有 Vitest UI 浏览器标签页

**经验教训**:
- 使用 `npx vitest run` 进行自动化测试
- 使用 `npx vitest --ui` 进行交互式测试
- 避免同时运行多个 Vitest UI 服务器
