const md = new MarkdownIt({
  html: true,
  linkify: true,
  typographer: true,
  highlight: function (str, lang) {
    if (lang && hljs.getLanguage(lang)) {
      return `<pre class="hljs"><code class="language-${lang}">${hljs.highlight(str, { language: lang }).value}</code></pre>`
    }
    return `<pre class="hljs"><code>${md.utils.escapeHtml(str)}</code></pre>`
  }
})

const getRelatedPosts = (id: string, category: string | undefined, limit: number = 5) => {
  if (!category) return []
  return posts.value
    .filter(post => post.id !== id && post.category === category)
    .sort((a, b) => new Date(b.publishedAt).getTime() - new Date(a.publishedAt).getTime())

## 🎯 关键技术决策

### 1. Markdown 渲染库选择

**决策**：使用 markdown-it 而不是 remark

**原因**：
- markdown-it 性能更好，渲染速度更快
- 插件生态更丰富，易于扩展
- 社区活跃，文档完善
- 与 highlight.js 集成更简单

### 2. 代码高亮库选择

**决策**：使用 highlight.js 而不是 shiki

**原因**：
- highlight.js 性能更好，渲染速度更快
- 支持更多语言（180+）
- 主题丰富，易于自定义
- 与 markdown-it 集成更简单

### 3. 目录生成方式

**决策**：使用 markdown-it-anchor 和 markdown-it-table-of-contents 插件

**原因**：
- 自动生成目录，无需手动维护
- 支持自定义样式和配置
- 性能优秀，生成速度快
- 易于扩展和维护

### 4. 主题系统支持

**决策**：使用 CSS 变量 + 动态 Class

**原因**：
- 与项目现有主题系统一致
- 易于维护和扩展
- 性能优秀，切换流畅
- 支持亮色和暗色主题

### 5. 响应式设计策略

**决策**：移动优先策略

**原因**：
- 符合现代 Web 开发最佳实践
- 提升移动端用户体验
- 易于维护和扩展
- 性能优秀
