# AI 错误记录与改进方案

> 创建时间：2026年3月1日
> 目的：记录 AI 在开发过程中犯下的错误，作为经验沉淀

---

## 错误记录

### 错误 #1: 忽视 MCP 工具的实际观察能力

**时间**：2026年3月1日

**错误描述**：
在优化项目时，一直通过读取代码文件来"猜测"UI问题，而没有使用 Playwright MCP 工具实际截图观察页面效果。

**影响**：
- 无法发现实际的 UI 渲染问题
- 误判优化效果
- 浪费时间在代码层面而忽略视觉层面

**正确做法**：
1. 使用 `browser_navigate` 访问页面
2. 使用 `take_screenshot` 截图实际观察
3. 使用 `browser_snapshot` 获取页面结构
4. 结合视觉和代码分析

**改进方案**：
- 任何涉及 UI 的任务，必须先用 MCP 工具实际观察
- 截图保存作为对比基准
- 使用 frontend-tester 验证 UI 变更

---

### 错误 #2: 任务执行缺乏视觉验证

**时间**：2026年3月1日

**错误描述**：
完成代码修改后只运行了 TypeScript 检查和构建，没有实际验证页面渲染效果。

**影响**：
- 博客页面内容为空的问题没有被发现
- UI 缺陷被遗漏

**正确做法**：
- 每次修改后使用 Playwright 截图验证
- 使用 frontend-tester agent 进行 UI 验证

---

### 错误 #3: 过度依赖代码分析，忽视用户体验视角

**时间**：2026年3月1日

**错误描述**：
从代码架构角度进行优化，但没有从用户视角审视整体 UI 设计。

**影响**：
- UI 设计缺陷被忽视
- 用户体验问题未被发现

**正确做法**：
- 先从用户视角截图审查
- 再从代码层面分析
- 最后结合两者制定方案

---

## 经验总结

1. **UI 项目必须先看再改**：截图 > 代码分析
2. **验证必须包含视觉验证**：构建通过 ≠ UI 正常
3. **多代理协作时明确分工**：让专业代理做专业的事
4. **记录错误避免重复**：每次犯错后记录沉淀

### 错误 #4: MCP 截图时未等待动态内容加载

**时间**：2026年3月1日

**错误描述**：
使用 Playwright MCP 截图时，在页面导航后立即截图，没有等待 Vue 应用的动态内容加载完成。导致截图显示空白页面，误判为"首页纯白不渲染"。

**影响**：
- 误判首页为"纯白色页面，评分1/10"
- 错误地认为存在严重的渲染问题
- 浪费时间排查不存在的 bug

**根本原因**：
- Vue SPA 应用需要时间挂载和渲染
- 页面导航后立即截图会捕获到空白状态
- 没有使用 `browser_wait_for` 等待内容加载

**正确做法**：
```javascript
// 1. 导航到页面
await page.goto('http://localhost:5173/')

// 2. 等待内容加载（关键步骤！）
await new Promise(f => setTimeout(f, 2000))  // 或使用 browser_wait_for

// 3. 验证内容已加载
const snapshot = await browser_snapshot()

// 4. 确认 main 区域有内容后再截图
await browser_take_screenshot()
```

**改进方案**：
- 建立"导航→等待→验证→截图"的标准流程
- 检查 `main` 区域是否有子元素作为内容加载标志
- 使用 `browser_wait_for` 的 `text` 参数等待特定文本出现
