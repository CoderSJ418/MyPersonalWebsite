## 一、产品概述

### 1.1 产品定位

MyPersonalWebsite 是一个基于 Vue 3 + Pinia + Vite 技术栈开发的个人博客与前端技术作品集一体化网站。核心价值在于面向同行开发者提供可复用的技术实践参考，同时向潜在雇主展示综合能力，兼顾普通读者的浏览体验，最终沉淀个人技术品牌。

### 1.2 产品愿景

打造一个集「技术展示、经验分享、风格定制」于一体的个人品牌平台，重点落地 AI 提示词工程、UI/UX 设计、前端工程化三大能力，为前端开发者提供高质量的实践参考。

### 1.3 角色定位

以「AI 提示词工程专家 + UI/UX 设计师 + 前端工程化实践者」三维视角定位项目，核心面向同行开发者输出可复用的技术方案。

## 二、目标用户

### 2.1 主要用户群体

| 用户群体 | 核心需求 | 使用场景 |
|---------|---------|---------|
| 潜在雇主/合作方 | 评估技术能力、项目落地能力、UI/UX 审美及工程化思维 | 浏览作品集案例、技术博客深度内容，评估招聘适配性 |

- 使用频率: 偶尔（招聘时）

- 年龄: 24-35 岁
- 职位: 前端工程师 / 全栈工程师
- 关注点: 技术细节、代码质量、设计思路、最佳实践
- 使用频率: 经常（学习参考）

**画像 3: 技术爱好者**
- 年龄: 20-40 岁
- 职位: 学生 / 转行者 / 技术博主
- 关注点: 学习资源、技术趋势、实用工具
- 使用频率: 定期（浏览学习）

## 三、功能需求

### 3.1 核心功能（MVP）

#### 3.1.1 博客模块

**优先级**: P0（最高）

**功能描述**:
- **文章列表展示**
  - 按发布时间倒序排列
  - 显示标题、摘要、发布时间、分类/标签
  - 支持分页加载（每页 10 篇）
  - 采用虚拟列表算法优化长列表渲染（≥50 篇）

- **分类与检索**
  - 支持按分类筛选文章（Vue3、TypeScript、工程化、设计等）
  - 支持按标签筛选（提示词工程、性能优化、调试技巧等）
  - 提供关键词搜索功能
  - 采用「关键词模糊匹配 + 标签权重排序」算法
  - 检索响应时间 ≤100ms

- **文章详情**
  - 代码块高亮展示（支持 Vue3、TS、JS、CSS 等前端语言）
  - 标注代码对应库版本（如 Vue 3.4.15）
  - 段落排版优化（行高 1.5，字号 16px）
  - 目录导航（长文自动生成）
  - 嵌入 Context7 拉取的官方文档引用链接
  - 支持代码复制功能
  - 支持文章分享（复制链接）

- **基础管理**
  - 后台支持文章发布、编辑、删除
  - 仅个人后台可见，前端不暴露编辑入口
  - 操作流程遵循多步任务拆解逻辑
  - 支持文章草稿保存

**验收标准**:
- [ ] 文章列表加载时间 ≤300ms
- [ ] 搜索功能准确率 ≥95%
- [ ] 代码块高亮正确率 100%
- [ ] 目录导航自动生成准确率 100%

#### 3.1.2 作品集模块

**优先级**: P0（最高）

**功能描述**:
- **作品展示**
  - 以卡片形式呈现
  - 包含作品封面、标题、技术栈标签（标注版本）、简短描述
  - 支持按技术栈分类筛选（Vue3、React、Node.js 等）
  - 响应式网格布局（桌面 3 列、平板 2 列、移动 1 列）

- **作品详情**
  - 展示项目背景、实现功能、核心技术亮点
  - 重点体现提示词工程应用、UI/UX 设计思路、工程化方案
  - 演示链接/源码链接（若有）
  - 附关键代码片段及 Context7 拉取的官方示例对照
  - 支持技术栈标签点击跳转相关作品

- **作品筛选与排序**
  - 支持按技术栈筛选
  - 支持按完成时间排序
  - 支持按项目复杂度排序

**验收标准**:
- [ ] 作品卡片加载时间 ≤200ms
- [ ] 筛选功能响应时间 ≤100ms
- [ ] 图片懒加载触发延迟 ≤50ms

#### 3.1.3 个人介绍模块

**优先级**: P1（高）

**功能描述**:
- **核心内容展示**
  - 个人经历（工作年限、教育背景）
  - 技术栈（明确版本范围，如 Vue 3.4.x、TS 5.3.x）
  - 项目经验（精选 3-5 个重点项目）
  - 联系方式（邮箱、GitHub、LinkedIn）
  - 突出 AI 提示词工程、UI/UX 设计、前端工程化三大核心能力

- **交互优化**
  - 简洁布局，信息层次清晰
  - 支持快速跳转至博客、作品集页面
  - 适配多终端展示
  - 无冗余交互

**验收标准**:
- [ ] 页面加载时间 ≤500ms
- [ ] 信息展示清晰，无视觉干扰
- [ ] 移动端适配良好

#### 3.1.4 调色板（风格定制）

**优先级**: P2（中，MVP 阶段简化）

**功能描述**:
- **配色方案切换**
  - 从 UI-UX-Pro-Max 内置 95 套配色方案中筛选 8 套适配场景
  - 配色方案：默认、深蓝、活力紫、极简灰、科技蓝、莫兰迪粉、商务灰、清新绿
  - 配色覆盖：主色、辅助色、中性色、文本色、背景色
  - 切换后即时生效，全局样式同步更新
  - 包含平滑过渡动画（0.3s）
  - 基于 localStorage 持久化用户偏好

- **UI 风格组件切换**
  - 从 UI-UX-Pro-Max 内置 57 种 UI 风格中选取核心风格
  - 风格列表：玻璃拟态、黏土拟态、扁平卡片、阴影卡片、渐变卡片、边框卡片、粗野主义、苹果风网格布局
  - 风格切换覆盖核心组件（博客卡片、作品卡片、按钮、面板、导航栏、代码块容器）
  - 与配色方案、字体组合深度联动

- **调色板交互面板**
  - 独立交互面板，包含配色选择、风格选择、字体组合选择器
  - 实时预览区域（展示博客卡片、按钮等核心组件）
  - 选中选项有明显激活状态标识
  - 面板支持收起/展开
  - 桌面端默认悬浮展示，移动端转为抽屉式或底部弹窗

**验收标准**:
- [ ] 配色切换响应时间 ≤100ms
- [ ] 风格切换响应时间 ≤100ms
- [ ] 过渡动画流畅（帧率 ≥60fps）
- [ ] localStorage 持久化成功

### 3.2 扩展功能（MVP 后迭代）

#### 3.2.1 AI 内容生成

**优先级**: P3（低，迭代功能）

**功能描述**:
- **辅助创作**
  - 集成 AI 工具，支持博客文章草稿生成
  - 技术要点提炼
  - 代码示例优化建议
  - 生成内容需结合 Context7 拉取的官方文档校验

- **风格适配**
  - 生成适配当前页面配色/风格的文章封面图
  - 生成作品展示文案
  - 保持视觉统一性

- **提示词模板**
  - 提供针对前端开发的提示词模板
  - 模板分类：组件生成、调试分析、架构设计
  - 供同行借鉴复用

**验收标准**:
- [ ] AI 生成内容准确率 ≥85%
- [ ] 生成时间 ≤5s
- [ ] 内容符合当前技术栈版本

#### 3.2.2 暗黑模式

**优先级**: P3（低，迭代功能）

**功能描述**:
- 与调色板配色联动
- 支持一键切换暗黑/亮色基础模式
- 适配不同浏览场景
- 自动检测系统主题偏好

**验收标准**:
- [ ] 暗黑模式切换流畅
- [ ] 所有页面适配暗黑模式
- [ ] 颜色对比度符合 WCAG 标准

#### 3.2.3 阅读统计

**优先级**: P3（低，迭代功能）

**功能描述**:
- 展示文章阅读量、作品访问量
- 匿名统计，不关联用户身份
- 遵循数据安全规范
- 不收集敏感信息

**验收标准**:
- [ ] 统计数据准确
- [ ] 隐私保护符合 GDPR 标准
- [ ] 不影响页面性能

## 四、非功能需求

### 4.1 性能需求

#### 4.1.1 核心性能指标

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 首屏加载时间 (LCP) | ≤1.8s | 最大内容绘制 |
| 首次内容绘制 (FCP) | ≤1.2s | 首次内容可见 |
| 累积布局偏移 (CLS) | ≤0.1 | 布局稳定性 |
| 交互到下一次绘制 (INP) | ≤200ms | 交互响应速度 |
| Lighthouse 评分 | ≥90 分 | 综合性能评分 |
| 风格切换响应时间 | ≤100ms | 主题切换 |
| 页面跳转响应时间 | ≤100ms | 路由切换 |
| 文章搜索加载 | ≤300ms | 搜索结果 |
| 图片懒加载延迟 | ≤50ms | 图片加载 |

#### 4.1.2 资源性能指标

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 静态资源压缩率 | ≥60% | JS/CSS 混淆压缩 |
| 图片格式优化 | WebP/AVIF | 现代图片格式 |
| 缓存命中率 | ≥80% | CDN + 浏览器缓存 |
| 代码分割 | 启用 | 按路由分割 |
| 虚拟列表 | ≥50 项 | 长列表优化 |

#### 4.1.3 算法优化目标

- **检索算法**: 关键词模糊匹配 + 标签权重排序，准确率 ≥95%，耗时 ≤100ms
- **渲染优化**: 组件懒加载 + 虚拟列表，帧率稳定 ≥60fps
- **资源加载**: 优先级加载 + 预加载，核心资源优先加载

### 4.2 安全需求

#### 4.2.1 数据安全

- XSS 防护：输入内容过滤转义
- CSRF 防护：Token 验证
- CSP 策略：限制第三方资源加载
- localStorage 仅存储用户主题偏好，不涉及敏感信息

#### 4.2.2 合规性

- 网站内容需拥有版权或授权，标注引用来源
- 适配 WCAG 2.1 基础可访问性标准
- 支持屏幕阅读器基本操作
- 部署环境默认开启 HTTPS 加密

#### 4.2.3 伦理约束

- AI 生成内容需标注来源，避免虚假信息传播
- 不收集、不滥用用户行为数据
- 对抗性提示防御：过滤诱导生成违规代码、版本不匹配内容的提示指令

### 4.3 兼容性需求

#### 4.3.1 浏览器兼容

| 浏览器 | 版本要求 |
|-------|---------|
| Chrome | 最新版 |
| Firefox | 最新版 |
| Edge | 最新版 |
| Safari | 14+ |
| 微信浏览器 | 最新版 |

#### 4.3.2 设备兼容

- 桌面端：1920×1080 及以上
- 平板端：768px - 1024px
- 移动端：≥320px
- 触控设备：iOS、Android

#### 4.3.3 技术兼容

- 基于 Vue 3 生态，兼容 Pinia、Vite 等核心工具
- 支持 UI-UX-Pro-Max 原生集成
- 样式兼容 CSS 变量、Flex/Grid 布局

### 4.4 可维护性需求

#### 4.4.1 代码规范

- 遵循 Vue 3 官方指南
- TypeScript 严格模式
- 驼峰式命名变量/函数
- PascalCase 命名组件
- 单组件代码量 ≤500 行
- 样式遵循 BEM 规范
- 注释覆盖率 ≥70%

#### 4.4.2 工程化规范

- ESLint + Prettier + Husky
- 提交前校验代码
- Git Flow 工作流
- 提交信息遵循 Conventional Commits
- 组件、核心函数需写文档注释

#### 4.4.3 样式模块化

- 配色、风格组件通过 CSS 变量、动态 Class 实现
- 便于扩展新配色/风格
- 符合组件化工程规范

### 4.5 可访问性需求

#### 4.5.1 键盘导航

- 所有交互元素可通过键盘访问
- Focus 状态清晰可见
- 支持 Tab 键导航

#### 4.5.2 ARIA 标签

- 语义化 HTML
- ARIA 标签完整
- 屏幕阅读器支持

#### 4.5.3 颜色对比度

- 符合 WCAG AA 标准（4.5:1）
- 不依赖颜色传达信息

### 4.6 测试需求

#### 4.6.1 单元测试

- 核心组件单元测试覆盖率 ≥80%
- 使用 Vitest 工具
- 自动化测试

#### 4.6.2 集成测试

- 关键流程端到端测试
- 使用 Playwright 工具
- 浏览器截图对比

#### 4.6.3 性能测试

- Lighthouse 定期审计
- 性能指标监控
- 性能回归检测

- UI 组件、交互效果贴合 UI-UX-Pro-Max 设计系统

- 备选 Cloudflare Pages

## 八、风险与约束

### 8.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| UI-UX-Pro-Max 集成复杂 | 高 | 中 | 提前调研，分阶段集成 |
| Context7 文档拉取不稳定 | 中 | 低 | 备用本地文档缓存 |
| 性能优化难度大 | 中 | 中 | 早期性能基准测试 |
| 第三方库版本兼容 | 低 | 低 | 严格版本锁定 |

### 8.2 业务风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| 内容更新不及时 | 中 | 中 | 建立内容更新流程 |
| 用户反馈收集困难 | 低 | 低 | 集成反馈系统 |
| SEO 效果不佳 | 中 | 中 | 持续 SEO 优化 |

### 8.3 时间风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| 开发周期超期 | 高 | 中 | 敏捷迭代，MVP 优先 |
| 测试时间不足 | 中 | 中 | 自动化测试覆盖 |
| 上线延期 | 中 | 低 | 提前部署演练 |
